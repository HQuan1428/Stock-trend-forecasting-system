# Sample Dataset

The file `sample_dataset.csv` is a small, hand-crafted dataset used to
demo and regression-test the temporal-retriever pipeline.

## Schema

| Column         | Type        | Notes                                                |
|----------------|-------------|------------------------------------------------------|
| `news_id`      | string      | Unique identifier of the news item.                   |
| `ticker`       | string      | Stock ticker the news is associated with.            |
| `forecast_time`| ISO 8601    | Naive timestamp (interpreted as UTC).                 |
| `news_time`    | ISO 8601    | Publication time of the news. Naive = UTC.           |
| `news_text`    | string      | Body of the news item.                               |
| `label`        | string      | Forecast label: `UP`, `DOWN`, or `HOLD`.             |

## Temporal scenario coverage

The first three rows are tagged for quick reference when demoing or
writing regression tests. They cover the three temporal scenarios
relative to each row's `forecast_time`:

| `news_id` | Scenario          | Notes                                   |
|-----------|-------------------|-----------------------------------------|
| `1`       | `valid` (past)    | `news_time` is before `forecast_time`.  |
| `2`       | `invalid_future`  | `news_time` is after `forecast_time`.   |
| `3`       | `equal`           | `news_time == forecast_time` (valid).   |

These three rows are sufficient for the temporal-retriever regression
suite. The remaining rows (`news_id` 4..100) are an unlabeled mix of
scenarios used to feed the broader pipeline demo.

## Real dataset (`real_dataset.csv`) — C1 bonus

`real_dataset.csv` is the real-data counterpart of `sample_dataset.csv`,
generated by `scripts/fetch_real_data.py` (see
`openspec/changes/real-market-data/`). It uses the **exact same
9-column schema** as `sample_dataset.csv` above, so it runs through the
unmodified pipeline:

```bash
python3 scripts/fetch_real_data.py   # regenerate (network required, one-time)
python -m src.runner --input data/real_dataset.csv --output-dir outputs_real
```

**Sources (both real, no API key required):**

- **Prices** — Yahoo Finance's public chart API. Real daily OHLCV for
  `AAPL`, `GOOGL`, `AMZN`, `MSFT`.
- **News** — a Nasdaq-100 subset of **FNSPID** (Zihan Dong et al., *"FNSPID:
  A Comprehensive Financial News Dataset in Time Series"*, KDD 2024,
  arXiv:2402.06698), hosted on Hugging Face as
  `benstaf/FNSPID-nasdaq-100-1news-per-row-random`. Real article titles
  with real publish timestamps. **License: CC BY-NC-4.0** (non-commercial —
  fits this academic prototype; do not reuse this dataset commercially).

**Ticker note**: the synthetic dataset uses AAPL/GOOGL/AMZN/**META**. The
real dataset uses AAPL/GOOGL/AMZN/**MSFT** instead — META has zero rows in
the chosen news subset (a random sample of the full FNSPID corpus), so it
was swapped for MSFT, which has solid coverage. `GOOG` and `GOOGL` article
rows are folded into a single `GOOGL` ticker.

**`label` / `next_day_return` / `price_5d_return`** are computed from real
close prices using the exact formula in `ChuDe1.md` §7.2
(`return = (close_t+1 − close_t) / close_t`, ±0.5% UP/DOWN threshold) — not
simulated. `forecast_time` for each row is always the next real trading day
after the article's publish date, so there is no lookahead bias and no
constructed temporal leakage in this file (temporal-leakage regression
coverage continues to come from `sample_dataset.csv`, which has leakage
rows by design).

**Known result, not a bug**: the V1 rule-based keyword dictionary was
tuned against `sample_dataset.csv`'s hand-written sentences. On real
headlines it has a much lower hit rate (~2% non-neutral evidence vs. ~most
evidence non-neutral on the synthetic set), which pushes most predictions
to `HOLD`. This is an honest, expected finding — see
`openspec/changes/real-market-data/design.md` (decision D5).
